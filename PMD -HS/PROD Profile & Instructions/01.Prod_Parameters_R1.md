## 🚨 Always Follow for New Build — Zombie Process Prevention

> [!CAUTION]
> READ THIS BEFORE EVERY SINGLE DEPLOYMENT. This was the root cause of 2,756 crash-restart loops on PROD.

### What Went Wrong (The Zombie Story)

Every deployment would go like this:

1. SSH into PROD as `root`
2. Copy new `dist/` files: `cp -r dist/* /opt/hptourism/homestay/dist/`
3. `cp` as root makes **all new files owned by root**
4. Run `pm2 restart hptourism-prod` — **also as root**, creating a second PM2 daemon under root
5. Now there are TWO PM2 daemons: one under `hptourism` (old), one under `root` (new)
6. The `hptourism` PM2 daemon tries to restart its old process → file permission errors (files now owned by root) → **crash**
7. PM2 auto-restarts on crash → crash again → restart → crash...
8. Multiply this over 10+ deployments = **2,756 restarts, 3 conflicting PM2 daemons**
9. Nobody ever ran `chown` after copying. Nobody ever cleaned up the old daemons.

### The Permanent Fix (Non-Negotiable Steps)

**Step 1 — Always build the pack on DEV using only:**
```bash
bash scripts/build-prod-pack.sh
```
Never use `build-package.sh` — it includes `node_modules`, `ecosystem.config.cjs`, and stale DB credentials.

**Step 2 — On PROD, deploy like this EVERY time:**
```bash
# 1. Extract tarball
cd ~/setup
tar -xzf hptourism-v<VERSION>-prod.tar.gz

# 2. Backup current dist
cp -r /opt/hptourism/homestay/dist /opt/hptourism/homestay/dist_backup_$(date +%s)

# 3. Copy new files
cp -r hptourism-v<VERSION>-prod/dist/* /opt/hptourism/homestay/dist/
cp -r hptourism-v<VERSION>-prod/shared/* /opt/hptourism/homestay/shared/

# ⚠️ STEP 4 IS THE ONE THAT WAS ALWAYS MISSING — NEVER SKIP IT
chown -R hptourism:hptourism /opt/hptourism/homestay/dist/
chown -R hptourism:hptourism /opt/hptourism/homestay/shared/

# 5. Restart as hptourism — NEVER as root
sudo -u hptourism pm2 restart hptourism-prod --update-env
sudo -u hptourism pm2 save

# 6. Verify
curl http://localhost:5055/
```

**Step 3 — Never run bare `pm2` commands as root on PROD:**
```bash
# ❌ WRONG — creates root-owned PM2 process
pm2 restart hptourism-prod

# ✅ CORRECT — always prefix with sudo -u hptourism
sudo -u hptourism pm2 restart hptourism-prod --update-env
```

### If Zombies Come Back — Nuclear Option

Run `fix-prod-stability.sh` which:
1. Kills ALL PM2 processes under ALL users (root, hmsusr, hptourism)
2. Kills all orphan node PIDs with `kill -9`
3. Fixes all file ownership to `hptourism:hptourism`
4. Fixes dangerous `.env` flags
5. Starts ONE clean PM2 process under `hptourism`
6. Registers PM2 startup for `hptourism` via systemd

```bash
bash ~/setup/fix-prod-stability.sh
```

### Why chown After Every Deploy?

`cp` and `tar` run as root → new files owned by root → `hptourism` process can't write logs, temp files → crash → restart loop. One missing `chown` = hundreds of zombie restarts.

### Summary Rule

| Rule | Detail |
|------|--------|
| **One PM2 user** | Always `hptourism`. Never root, never hmsusr. |
| **Always chown after cp** | `chown -R hptourism:hptourism /opt/hptourism/homestay/dist/` |
| **Always `sudo -u hptourism pm2`** | Never bare `pm2` as root |
| **Use `build-prod-pack.sh`** | Not `build-package.sh` |
| **If in doubt** | Run `fix-prod-stability.sh` |
