## Deployment Notes & Lessons Learned

### SOP Reminder: Always Run `db:push`

The v1.3.5 → v1.3.6 transition exposed a critical gap in the deployment SOP:

> **Schema changes in Drizzle ORM are NOT automatically applied to PROD.**
> Even if no migrations are explicitly planned, always run `db:push` after deploying new code to catch any schema additions.

**Recommended addition to SOP Step 4:**
```
# After copying code files:
cd /opt/hptourism/homestay/
sudo -u hptourism bash -c "export $(grep -v '^#' .env | grep -v '^\s*$' | xargs) && npm run db:push"
```

### Version Sync Fix

The version display issue recurred across multiple releases because two files contained version numbers:
- `package.json` (source of truth)
- `client/src/config/version.ts` (was hardcoded, now auto-reads)

**As of v1.3.6, only `package.json` needs to be updated.** The UI automatically picks up the version at build time.
