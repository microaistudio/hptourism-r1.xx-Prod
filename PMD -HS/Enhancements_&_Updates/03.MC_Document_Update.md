# MC Zone — Additional Document Requirements

## Problem

Properties located in **Municipal Corporation (MC)** areas require additional regulatory documents that are not currently collected in the application form. The current Step 5 (Document Upload) shows the same document set regardless of location type.

### Location Types in System

| Code | Full Name | Extra Documents Needed? |
|------|-----------|------------------------|
| `mc` | Municipal Corporation | ✅ Yes — Building Plan, MC NOC |
| `tcp` | Town & Country Planning | ❌ No (existing docs sufficient) |
| `gp` | Gram Panchayat | ❌ No (existing docs sufficient) |

The `location_type` field is already captured in the application form (Step 2 — Property Details) and stored in the database.

---

## Current Document List (Step 5)

All applicants currently upload:

| # | Document | Required |
|---|----------|----------|
| 1 | Revenue Papers (Jamabandi / Registry) | ✅ Mandatory |
| 2 | Affidavit (Section 29) | ✅ Mandatory |
| 3 | Undertaking (Form C) | ✅ Mandatory |
| 4 | Commercial Electricity Bill | Conditional (if applicable) |
| 5 | Commercial Water Bill | Conditional (if applicable) |
| 6 | Property Photos | ✅ Mandatory |
| 7 | Additional Documents | Optional |

---

## Proposed Changes

### New MC-Specific Documents

When `locationType === 'mc'`, show 2 additional mandatory upload slots:

| # | Document | When Shown | Required |
|---|----------|-----------|----------|
| 8 | **Approved Building Plan** | MC zone only | ✅ Mandatory |
| 9 | **MC NOC (No Objection Certificate)** | MC zone only | ✅ Mandatory |

> [!NOTE]
> These documents appear **only when the applicant's property is in an MC area**. TCP and GP applicants see the existing document list unchanged.

## Implementation

### Files to Modify

#### 1. `client/src/components/applications/form-sections/Step5Documents.tsx`
- Add `locationType` to props interface
- Add `mcBuildingPlan` and `mcNoc` fields to `UploadedDocuments` interface
- Add conditional MC section with two new `ObjectUploader` components
- Show section only when `locationType === 'mc'`
- Pattern already exists: follows same approach as `requiresCommercialUtilityProof`

#### 2. `client/src/pages/applications/new.tsx`
- Pass `locationType` prop to `Step5Documents`
- Initialize `mcBuildingPlan` and `mcNoc` in uploaded documents state
- Include MC documents in form submission payload
- Add validation: if MC zone, both documents must be uploaded before submission

#### 3. `server/db-storage.ts` (Document Sync)
- Add `mc_building_plan` and `mc_noc` to document sync mapping
- Ensure MC documents are synced from JSONB to documents table

#### 4. DA Scrutiny View (already dynamic)
- No changes needed — DA scrutiny already renders all documents from the documents table dynamically
