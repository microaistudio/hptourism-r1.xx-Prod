# Deployment Standard Operating Procedure (SOP)

> **⚠ CRITICAL: ZOMBIE PROCESS PREVENTION**
> Never deploy by simply copying files as `root` and running `pm2 restart`.
> This caused 3,000+ zombie restarts due to file ownership conflicts.
> **You must follow every step in this document.**

---

## PROD Server Parameters

| Parameter | Value |
| :--- | :--- |
| **App User** | `hptourism` (NEVER `root` or `hmsusr`) |
| **Port** | `5055` (Nginx proxies `:443` → `:5055`) |
| **PM2 Process Name** | `hptourism-prod` |
| **PM2 Command Prefix** | `sudo -u hptourism pm2 <cmd>` |
| **Base Directory** | `/opt/hptourism/homestay/` |
| **Database** | `hptourism` on PostgreSQL 16 |

---

## Step 1: Build the Package (On DEV Machine)

```bash
cd ~/Projects/hptourism/homestay/
bash scripts/build-prod-pack.sh
```

Output: `~/Projects/setup/Install_packs/hptourism-v<VERSION>-prod.tar.gz` (~91MB, secure)

This script automatically:
- Runs `npm run build`
- Excludes `.env`, `ecosystem.config.cjs`, `node_modules`, `Database/db-config.env`
- Scans for leaked dev URLs

---

## Step 2: Upload to PROD

```bash
scp ~/Projects/setup/Install_packs/hptourism-v<VERSION>-prod.tar.gz root@homstyshmapp:~/setup/
```

---

## Step 3: Deploy on PROD (SSH as root)

```bash
ssh root@homstyshmapp
cd ~/setup
```

### 3a. Extract
```bash
tar -xzf hptourism-v<VERSION>-prod.tar.gz
```

### 3b. Backup Current Version
```bash
cp -r /opt/hptourism/homestay/dist /opt/hptourism/homestay/dist_backup_$(date +%s)
```

### 3c. Copy New Files
```bash
cp -r hptourism-v<VERSION>-prod/dist/* /opt/hptourism/homestay/dist/
cp -r hptourism-v<VERSION>-prod/shared/* /opt/hptourism/homestay/shared/
cp hptourism-v<VERSION>-prod/package.json /opt/hptourism/homestay/package.json
cp -r hptourism-v<VERSION>-prod/migrations/* /opt/hptourism/homestay/migrations/ 2>/dev/null || true
```

### 3d. ⚠ FIX OWNERSHIP (THE MOST CRITICAL STEP)
```bash
chown -R hptourism:hptourism /opt/hptourism/homestay/
```
> **If you skip this, PM2 will crash-loop with `EACCES permission denied` errors.**

### 3e. Run Database Migrations
```bash
cd /opt/hptourism/homestay/
sudo -u hptourism bash -c "export \$(grep -v '^#' .env | grep -v '^\s*$' | xargs) && npm run db:push"
```

### 3f. Restart PM2
```bash
sudo -u hptourism pm2 restart hptourism-prod --update-env
sudo -u hptourism pm2 save
```

### 3g. Verify
```bash
sudo -u hptourism pm2 list          # Restarts should be 0
curl -I http://localhost:5055/       # Should return HTTP 200
```

## Emergency: Nuclear Zombie Cleanser

If someone ran PM2 as root and the server is stuck in crash loops:

```bash
#!/bin/bash
# Kill ALL PM2 daemons across ALL users
pm2 stop all 2>/dev/null; pm2 delete all 2>/dev/null
sudo -u hmsusr pm2 stop all 2>/dev/null; sudo -u hmsusr pm2 delete all 2>/dev/null
sudo -u hptourism pm2 stop all 2>/dev/null; sudo -u hptourism pm2 delete all 2>/dev/null

# Kill orphaned Node processes
pgrep -f "node.*dist/index.js" | xargs kill -9 2>/dev/null || true
fuser -k 5055/tcp 2>/dev/null || true

# Fix ownership
chown -R hptourism:hptourism /opt/hptourism/homestay/

# Start fresh
sudo -u hptourism bash -c "cd /opt/hptourism/homestay/ && pm2 start ./dist/index.js --name hptourism-prod --cwd /opt/hptourism/homestay/ --max-memory-restart 1G"
sudo -u hptourism pm2 save
```
